# IoT-fiware 要件定義

## システム概要

FIWARE Orion をデータハブとして、PiCar-X・スマートフォン・スマートホームデバイスなど
各種 IoT 機器のデータを収集・活用するホーム IoT プラットフォーム。

LINE を主な UI として利用し、Google Home Mini への音声出力、カメラ撮影、位置情報記録などを実現する。

---

## 機能要件

### F1: LINE → Google Home 読み上げ ✅ 実装済み

| 項目 | 内容 |
|------|------|
| トリガー | LINE メッセージ送信（! 以外のテキスト） |
| 処理 | Google Home Mini (192.168.3.11) で TTS 読み上げ |
| 用途 | 外出先から猫への声かけ、自宅への伝言 |

### F2: LINE → Google Home 音量コントロール ✅ 実装済み

| コマンド | 動作 |
|----------|------|
| `!音量50` / `!ボリューム50` | 音量を 50% に設定（0〜100） |
| `!音量+` / `!ボリューム+` | 音量を 10% アップ |
| `!音量-` / `!ボリューム-` | 音量を 10% ダウン |
| `！音量XX` | 全角「！」も対応 |

### F3: LINE → PiCar-X カメラ撮影・返信 ✅ 実装済み（Token 要設定）

| 項目 | 内容 |
|------|------|
| トリガー | LINE メッセージ `!写真` |
| 処理 | PiCar-X カメラで撮影 → 写真を LINE に返信 |
| 副作用 | FIWARE Orion の PiCarX:001 エンティティに photo_url を記録 |
| 未設定 | LINE Channel Access Token（取得・設定が必要） |

### F4: スマートフォン GPS トラッキング ✅ 実装済み

| 項目 | 内容 |
|------|------|
| UI | ブラウザ（HTTPS 必須） `/gps/` |
| 操作 | スタート / エンド ボタンで区間記録 |
| 送信間隔 | 5 秒ごとに位置情報を POST |
| 保存先 | FIWARE Orion → Draco → PostgreSQL (`gpsroute.routes`) |
| 可視化 | Grafana GPS ルートトラッカーダッシュボード |
| 用途 | 移動ルートの記録・除排雪ルートなどの実測データ取得 |

### F4-B: GPS 連携写真機能 ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 写真アップロード | GPS 画面から撮影・選択し、位置情報（EXIF 優先 → トラッキング位置 → 現在地）とともに投稿 |
| 地図表示 | 撮影位置に 📷 マーカーを表示。自分の写真＝オレンジ、他デバイス＝青で色分け |
| 表示切替 | 「自分のみ」「全員分」をワンタップで切替 |
| 写真ポップアップ | マーカータップで写真・星評価（1〜5）・撮影日時を表示 |
| 星評価 | ポップアップ内で星をタップして評価し「★保存」で確定。本人のみ変更可、他デバイスは参照のみ |
| コメント機能 | 各写真にチャット形式でコメントを投稿・閲覧（デバイス間共有）。投稿者名（authorName）を表示 |
| コメント表示 | ポップアップ外の固定ボタン「💬 コメントを見る・書く」を押して開く（自動表示なし） |
| 写真ポップアップ位置 | 画像ロード完了後に自動で位置補正（ヘッダー等に隠れないよう panBy） |
| デバイス識別 | 初回アクセス時に名前＋4桁PINを入力。FNV-1a(名前:PIN) のハッシュを deviceId として使用。同名でも PIN が異なれば別デバイス扱い |
| デバイス設定変更 | 写真枚数エリア長押し → PIN 確認 → ①名前/PIN 変更（既存写真の引き継ぎ可）または②旧 ID の写真を番号選択で引き継ぎ |
| データ永続化 | Docker Volume（`iot_fiware_gps_data`）の `photos-data.json` に保存 |

### F5: 画像 AI 解析 〇 実装予定

| 項目 | 内容 |
|------|------|
| ハードウェア | Mac mini（メモリ 16GB）購入・出荷待ち |
| LLM | Gemma 4（ローカル実行） |
| 処理フロー | `!写真` → PiCar-X 撮影 → ローカル LLM → 解析結果を LINE 返信 |
| 判定内容 | 猫の有無、状態（寝ている・活動中など） |

### F6: スマートホーム制御 〇 実装予定

| コマンド（案） | 動作 |
|----------------|------|
| `!電気オン` | Smart Life/Tuya 照明をオン |
| `!電気オフ` | Smart Life/Tuya 照明をオフ |
| 用途 | 撮影前に照明を点灯、就寝前の消灯確認など |

### F7: PiCar-X 移動制御 〇 実装予定

| コマンド（案） | 動作 |
|----------------|------|
| `!前進` / `!後退` | PiCar-X を前後移動 |
| `!左` / `!右` | 旋回 |
| 用途 | 猫のいる場所まで移動して撮影 |

## 非機能要件

| 項目 | 内容 |
|------|------|
| 外部アクセス | Cloudflare Tunnel（PC 起動毎に URL 変更、固定ドメインなし） |
| ネットワーク | LAN: 192.168.3.0/24、Windows PC: 192.168.3.6 |
| データ永続化 | PostgreSQL（fiware-base プロジェクトで管理） |
| 可視化 | Grafana（grafana-fiware プロジェクトで管理） |
| コンテナ | Docker Desktop（Windows）、fiware-base-net 共有ネットワーク |
| PiCar-X | Raspberry Pi、192.168.3.17、Flask サーバー（ポート 5000） |
| Google Home | Google Home Mini、192.168.3.11 |
